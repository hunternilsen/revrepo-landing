1

Open the App in Domo

Find The Quartermaster on your Domo page. Your name loads automatically in the top-right corner via the Domo User API — no login, no configuration needed.

2

Browse the Tools Catalog

The Tools tab lists all mandatory external tools — Salesforce, Apollo.io, Gong, Slack, and more. Each card shows the tool name, description, and a direct “Visit Tool” link. Click any card to open the tool.

3

Get Access Info Two Ways

Click the (i) icon on any tool card to see how to request access. Toggle between Tooltip mode (floating overlay) and Expandable mode (inline card expansion) using the buttons in the header.

4

Check Today’s Lunch

Switch to the Lunch tab to see the daily cafeteria menu, fetched live from a Domo dataset. Items are grouped by category (Main Course, Side Dish, Dessert, etc.) with icons for quick scanning.

5

Track Onboarding Progress

A checklist popup (bottom-right corner) tracks key setup tasks for new team members. It starts minimized — click the button to expand it. State persists for the current session.

Platform

Built on Domo Pro-Code

Domo Pro-Code apps are custom HTML/JS/CSS applications that run inside Domo’s environment with direct access to platform APIs.

Vanilla JavaScript

No framework, no build step, no bundler. All logic lives in a single app.js file. Open the source and you see exactly what runs — nothing hidden, nothing compiled.

Tailwind CSS v4 via CDN

Styling is applied via the Tailwind v4 browser CDN — utility classes directly in HTML, no PostCSS, no tailwind.config.js, just copy-paste.

Domo Data API

The lunch menu is fetched live from a mapped Domo dataset using domo.get('/data/v1/lunchMenu'). The dataset alias is declared in manifest.json.

Deploy with Domo CLI

Local development via any HTTP server (no proxy needed for static content). Production deploys with domo publish — no CI/CD pipeline, no Docker, no cloud config.

Architecture

Two Files. That’s It.

The entire app lives in two files. No build step, no bundler, no dependencies to install.

index.html App Shell
HTML structure, Tailwind CSS via CDN, CSS variables, ryuu.js script tag, and all UI markup for tools, help, and lunch tabs.
app.js All Logic
Tab switching, tooltip/expandable modes, access info lookup, lunch menu fetch & render, user display, onboarding checklist state.
manifest.json Config
Domo app ID, dataset mappings (lunchMenu alias), scopes (appdb), and display settings (fullpage: true).

App thumbnail shown in Domo Design Studio. Required for publishing — Domo rejects the publish if missing.
